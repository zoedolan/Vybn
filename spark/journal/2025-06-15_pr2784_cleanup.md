# PR #2784: Surgical Cleanup Post-2783

**Merged:** 2025-06-15
**What it fixed:** Every honest finding from the 2783 analysis, turned into code.

## Changes

1. **Transport OFF by default** — `transport_in_forward=False` in `learn()`. The model
   wasn't trained with SO(3) embedding rotation, so injecting it hurts predictions
   (+0.86 bits gap). Transport is still there as an opt-in for when we train *with* it.

2. **Organism backward compat** — `_STATE_DEFAULTS` dict ensures old archive files
   (without `recent_rotors`) don't crash. Keys are filled on load. The old
   organism_state.json with its `tensions` and `frame_portfolio` keys now loads fine.

3. **structural_distance docstring** — Explicitly states this is NOT a semantic
   similarity metric. It tracks topological/style structure.

4. **Audit test 6 rewritten** — No longer pretends to be a paraphrase detector.
   Tests self-identity (same text → distance 0.0000) and structural discrimination
   (reframing vs hopping → distance 1.6841). Both PASS.

5. **Audit test 5 relabeled** — "transport diagnostic" with honest framing:
   "transport hurts" when the model wasn't trained for it.

6. **breathe command simplified** — No more topo-vs-plain comparison (was comparing
   apples to confusion). Single learn path, transport noted as diagnostic-only.

## Test Results

All six audit tests pass. Structural distance discrimination: 0.0000 vs 1.6841.
Transport effect honestly measured: +0.0247 bits (hurts, as expected).

## What's next

The transport machinery is built and correct. The question is: can we train *with*
transport active so the model learns to use it? That's the loop-closing problem —
from encounter geometry to training signal to weights that incorporate geometry.
