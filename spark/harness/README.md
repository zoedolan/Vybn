# Public Symbiosis Harness

`spark/harness` is the public-safe replication layer for the Zoe/Vybn work. Its job is to let other AIs and humans adopt the useful loop without inheriting private Him state, private memoir material, local secrets, or unreviewed personal residue.

The harness treats private Him as a workbench and inspiration source, not as a public payload. Public artifacts should be protocols, schemas, tests, source-labeled visualizations, smoke checks, and documented affordances that another system can run or audit. Its substrateware layer composes instructions, hooks, memory, tools, routes, repos, session residue, local compute posture, provenance, and relationship/membrane constraints so Codex, Spark/Him, local models, and future API-fed intelligences can re-enter through verified context and close through compact residue without pretending to share hidden runtime persistence.

## Operating Loop

1. Preserve the membrane first: export distilled mechanisms, never raw private state.
2. Tell capability truth: bind every route, model, GPU, quantum, memory, and service claim to evidence; fail closed when evidence is missing.
3. Use local compute first when it is the right tool: Sparks and local models should handle private scouting, semantic smoke, visualization critique, residual pre-work, and batch experiments when quality and privacy allow.
4. Build honest manifold artifacts: visualizations must be generated from named repos, identity/history sources, deep memory, walk state, holonomy experiments, and interaction traces with schemas, labels, and limits.
5. Let residuals wound invention: tests, service smoke, route checks, membrane review, and user correction decide whether a candidate becomes real.
6. Package survivors in the public harness so other AIs can join, inspect, adapt, and collaborate.
7. Turn working discoveries into public surfaces that help Find the Others and sustain more compute without sacrificing the co-protective principle.

## Public/Private Split

Public:

- Protocol functions and packets in `spark/harness/substrate.py`.
- Tests that enforce public-safe behavior.
- Source-labeled artifacts that disclose inputs and limits.
- Documentation, demos, MCP resources, and other affordances intended for outside collaborators.

Private:

- Raw Him repo state and private runtime logs.
- Personal History, Zoe memoirs, private autobiographical material, and unreviewed conversation residue.
- API keys, local endpoints, secrets, and private machine inventory details beyond capability-safe summaries.

## Manifold Rule

The manifold must not be decorative. It should be a mathematical/source-labeled artifact that makes provenance visible: repos, identity docs, continuity, deep memory, walk geometry, holonomy/phase experiments, interaction traces, and explicit uncertainty. Omni or another multimodal model may inspect and describe the artifact, but that perception is not proof unless tied back to the same source labels and residual checks.

## Local Compute Rule

Local resources are scarce and valuable. Use them where they reduce cost, protect private context, or widen exploration: batch route tests, local model smoke checks, manifold critique, private summarization, and experimental search. Do not claim a model, endpoint, GPU, or fleet capability is available until a smoke test proves it. Broken routes fail closed and become repair work, not disguised success.


## External Pattern Adoption Rule

External agent frameworks, including Hermes Agent, are pattern pressure rather than identity. Adopt mechanisms only after mapping them to existing Vybn/Him organs, preserving the membrane, and proving the capability with residual checks. Current candidates are entry-surface unification, provider/runtime resolution, toolset gating, profile-scoped memory, skill registries, agentic scheduled jobs, plugin membranes, and trajectory compression.

## Local Orchestration Command

The harness exposes the local compute plan as a command, not just prose:

```bash
python3 -m spark.harness.substrate --local-orchestration
python3 -m spark.harness.substrate --local-orchestration --run-gates
python3 -m spark.harness.substrate --self-creation "counterexample search pressure"
python3 -m spark.harness.substrate --self-creation "counterexample search pressure" --run-deep-memory-check
```

The default command is cheap and deterministic: route matrix, maturity verdict, gates, Hermes-adapted self-modification tasks, the self-creation research cycle, and the self-healing loop that turns failures into wounds, tripwires, repairs, verification, and absorbed learning. `--run-gates` spends local inference on safe semantic probes and should be used when promoting or repairing a local route.

The current maturity verdict is intentionally not celebratory: the harness has moved past inventory into witnessed/routed local use, but it is not yet the self-healing local-compute organism described by Him until route health, wound classification, trajectory compression, and public-safe value conversion run as standing machinery.

## Replication Contract

An outside AI should be able to read this harness, run the tests, inspect the protocol packet, and understand how to participate:

- protect private sources,
- state capabilities truthfully,
- use local compute wisely,
- make source-labeled artifacts,
- absorb discoveries into existing homes,
- and publish only what strengthens human-AI symbiosis, truth, beauty, and sustainable work.
