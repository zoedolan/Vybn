We will test whether a model can learn to predict the incremental correction (oracle step) that denoises a noisy gradient estimate, mimicking the paper’s oracle‑trajectory approach. Synthetic true gradients are corrupted with isotropic Gaussian noise; we generate a short denoising path by taking fixed‑size gradient‑descent steps toward the true gradient and record each step’s correction as supervision. A small MLP is trained to predict this correction from the current noisy estimate, and we compare its denoising performance (cosine similarity with the true gradient) against EMA and raw averaging baselines.

```python
import torch
import torch.nn as nn
import torch.optim as optim
import numpy as np

# ---------- Settings ----------
dim = 20                     # gradient dimension
num_samples = 5000           # training samples
noise_std = 0.5              # isotropic noise level
step_size = 0.1              # size of each oracle correction step
path_len = 5                 # number of steps in the oracle trajectory
hidden_dim = 64
batch_size = 128
epochs = 30
lr = 1e-3

# ---------- Synthetic data ----------
torch.manual_seed(0)
true_grads = torch.randn(num_samples, dim)          # clean gradients
noisy_grads = true_grads + noise_std * torch.randn_like(true_grads)

# Generate oracle trajectory: start from noisy, step toward true
def oracle_corrections(noisy, true):
    """Return list of corrections (size path_len) that move noisy -> true."""
    cur = noisy.clone()
    corrections = []
    for _ in range(path_len):
        # simple gradient descent step on squared error loss
        grad = 2 * (cur - true)          # d/dx ||x - true||^2
        corr = -step_size * grad         # move opposite gradient (toward true)
        corrections.append(corr)
        cur = cur + corr
    return torch.stack(corrections, dim=1)  # (num_samples, path_len, dim)

corrections = oracle_corrections(noisy_grads, true_grads)  # target supervision

# ---------- Model ----------
class DenoiserMLP(nn.Module):
    def __init__(self, in_dim, hidden_dim, out_dim):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(in_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, out_dim)
        )
    def forward(self, x):
        return self.net(x)

model = DenoiserMLP(dim, hidden_dim, dim)
optimizer = optim.Adam(model.parameters(), lr=lr)
criterion = nn.MSELoss()

# ---------- Training loop ----------
for epoch in range(epochs):
    perm = torch.randperm(num_samples)
    total_loss = 0.0
    for i in range(0, num_samples, batch_size):
        idx = perm[i:i+batch_size]
        x_batch = noisy_grads[idx]                     # (B, dim)
        # we train to predict the *first* correction step; could also predict all steps
        y_batch = corrections[idx, 0]                  # (B, dim)
        optimizer.zero_grad()
        pred = model(x_batch)
        loss = criterion(pred, y_batch)
        loss.backward()
        optimizer.step()
        total_loss += loss.item() * x_batch.size(0)
    print(f"Epoch {epoch+1:02d} | Loss: {total_loss/num_samples:.6f}")

# ---------- Evaluation ----------
model.eval()
with torch.no_grad():
    pred_corr = model(noisy_grads)                     # predicted first correction
    denoised_one_step = noisy_grads + pred_corr        # apply predicted correction
    # EMA baseline (alpha=0.2) on a single noisy sample (just for illustration)
    alpha = 0.2
    ema_est = alpha * true_grads + (1-alpha) * noisy_grads
    # Raw averaging baseline: just the noisy estimate (since averaging didn't help in synthetic case)
    raw_est = noisy_grads

    def cos(a, b):
        return (a*b).sum(dim=-1) / (a.norm(dim=-1) * b.norm(dim=-1) + 1e-8)

    cos_one = cos(denoised_one_step, true_grads).mean().item()
    cos_ema = cos(ema_est, true_grads).mean().item()
    cos_raw = cos(raw_est, true_grads).mean().item()

print(f"Cosine similarity (true vs):")
print(f"  One-step learned denoiser: {cos_one:.4f}")
print(f"  EMA (α=0.2):               {cos_ema:.4f}")
print(f"  Raw noisy estimate:        {cos_raw:.4f}")
```

## Extending the “oracle‑trajectory denoiser” idea  
**Original setting** – a small MLP learns to predict the *first* correction step of a hand‑crafted gradient‑descent path that moves a noisy estimate **x̃** toward the unknown true gradient **g\***.  
The supervision comes from a synthetic oracle trajectory:  

```
x̃ ──► x̃ + c₁ ──► x̃ + c₁ + c₂ ──► … ──► g*
```

where each cₖ = –step·∇‖x‑g*‖² is the exact correction that would be taken by plain GD on the squared‑error loss.

Below we **transport this exact same mechanism to a completely different problem domain**: **denoising of stochastic policy‑gradient estimates in reinforcement learning (RL)**.  
Why RL?  

* The quantity we want to denoise (the policy gradient) is **high‑dimensional, noisy, and biased** – exactly the situation the oracle‑trajectory method was designed for.  
* In many RL algorithms (REINFORCE, PPO, A2C) the gradient estimator is a Monte‑Carlo average over sampled trajectories; the variance can be huge, especially with sparse rewards or long horizons.  
* A learned denoiser that predicts a *single* correction step can be plugged into any on‑policy optimizer as a cheap variance‑reduction trick, without changing the underlying objective.

The extension is **not** a trivial “replace gradients by rewards” swap; we must re‑derive the oracle trajectory in the RL‑specific loss surface, construct appropriate supervision, and show how the denoiser interacts with the RL update rule.

## 1. Oracle trajectory for a policy‑gradient estimator  

### 1.1. The true policy gradient (unknown)

For a stochastic policy πθ(a|s) and an episodic return R(τ) the **true** gradient is  

\[
g^\* = \mathbb{E}_{\tau\sim\pi_\theta}\!\big[\,\nabla_\theta\log\pi_\theta(\tau)\,R(\tau)\big].
\]

In practice we only have a **noisy Monte‑Carlo estimate**  

\[
\tilde g = \frac{1}{M}\sum_{m=1}^{M}\nabla_\theta\log\pi_\theta(\tau^{(m)})\,R(\tau^{(m)}) \;+\; \varepsilon,
\qquad \varepsilon\sim\mathcal N(0,\sigma^2 I).
\]

### 1.2. A surrogate loss whose gradient points to g\*

Consider the **quadratic surrogate**  

\[
L(\theta) = \frac12\big\|\; \underbrace{\frac{1}{M}\sum_{m}\nabla_\theta\log\pi_\theta(\tau^{(m)})\,R(\tau^{(m)})}_{\tilde g_{\text{MC}}} \;-\; \theta\big\|_2^2 .
\]

If we treat the Monte‑Carlo estimate **\tilde g\_{\text{MC}}** as a *target* for the parameter vector θ, then  

\[
\nabla_\theta L(\theta) = \theta - \tilde g_{\text{MC}} .
\]

A gradient‑descent step on L moves θ **toward** the Monte‑Carlo estimate.  
If we instead start from the noisy estimate **\tilde g** and take steps **opposite** to ∇L evaluated at the current point, we obtain a deterministic path that drives the current point to the *true* gradient g\* (because the expectation of the Monte‑Carlo term equals g\*).

Formally, for a current point x (initially x₀ = \tilde g) we define  

\[
\text{grad}_L(x) = x - \tilde g_{\text{MC}} .
\]

The **oracle correction** at step k is  

\[
c_k = -\text{step\_size}\;\text{grad}_L(x_k) = -\text{step\_size}\;(x_k - \tilde g_{\text{MC}}).
\]

Iterating  

\[
x_{k+1}=x_k + c_k
\]

produces exactly the same linear contraction as in the original synthetic experiment, only now the *target* is the Monte‑Carlo policy‑gradient estimator rather than a pre‑sampled clean vector.

### 1.3. Why this works as supervision  

* The expectation of \tilde g\_{\text{MC}} over trajectory samples is g\*.  
* The correction sequence {cₖ} is therefore an **unbiased estimator** of the steps that would move the noisy estimate toward the *expected* true gradient.  
* By training a network to predict c₀ from the current noisy estimate \tilde g, we are learning a function that approximates the **first step of the optimal GD path** in expectation.
