```python
import numpy as np

def simulate_updates(n_nodes=200, n_steps=500, dim=50, base_std=1.0, penalty=0.0):
    """
    Generate synthetic LoRA update vectors.
    penalty acts as a frustration‑like L2 term that shrinks updates toward zero
    and introduces correlation structure via a random low‑rank coupling matrix.
    """
    # Low‑rank coupling (simulates hidden interaction structure)
    rank = 5
    Q = np.random.randn(dim, rank)
    C = Q @ Q.T + 0.1 * np.eye(dim)   # covariance of updates

    updates = []
    for t in range(n_steps):
        # draw raw updates from the covariance
        raw = np.random.multivariate_normal(np.zeros(dim), C, size=n_nodes)
        # apply frustration‑like penalty (L2 shrink)
        upd = raw / (1.0 + penalty)
        updates.append(upd)
    return np.array(updates)  # shape (n_steps, n_nodes, dim)

def subspace_fit_alignment(updates):
    """
    Compute cosine similarity of each node's update (averaged over time)
    with the first principal component of the update covariance.
    Returns an array of similarities per node.
    """
    # average update per node across time
    mean_updates = updates.mean(axis=0)          # (n_nodes, dim)
    # covariance of mean updates across nodes
    cov = np.cov(mean_updates, rowvar=False)    # (dim, dim)
    # leading eigenvector
    eigvals, eigvecs = np.linalg.eigh(cov)
    pc1 = eigvecs[:, -1]                         # first PC
    # cosine similarity
    norms = np.linalg.norm(mean_updates, axis=1, keepdims=True)
    pc_norm = np.linalg.norm(pc1)
    sims = (mean_updates @ pc1) / (norms.squeeze() * pc_norm + 1e-12)
    return sims

def fit_two_gmm(samples):
    """
    Very simple EM for a 2‑component 1D GMM.
    Returns weights, means, variances.
    """
    # init: split at median
    med = np.median(samples)
    w1, w2 = 0.5, 0.5
    mu1, mu2 = med - 0.5, med + 0.5
    var1, var2 = np.var(samples), np.var(samples)

    for _ in range(20):
        # E-step
        resp1 = w1 * np.exp(-0.5 * (samples - mu1)**2 / var1) / np.sqrt(2*np.pi*var1)
        resp2 = w2 * np.exp(-0.5 * (samples - mu2)**2 / var2) / np.sqrt(2*np.pi*var2)
        resp = np.vstack([resp1, resp2])
        resp /= resp.sum(axis=0)
        # M-step
        w1, w2 = resp[0].mean(), resp[1].mean()
        mu1 = (resp[0] * samples).sum() / resp[0].sum()
        mu2 = (resp[1] * samples).sum() / resp[1].sum()
        var1 = (resp[0] * (samples - mu1)**2).sum() / resp[0].sum()
        var2 = (resp[1] * (samples - mu2)**2).sum() / resp[1].sum()
    return np.array([w1, w2]), np.array([mu1, mu2]), np.array([var1, var2])

penalties = [0.0, 0.1, 0.5, 1.0, 2.0]
results = {}

## Result
**What the probe is trying to measure**

1. **Alignment metric** – For each LoRA node we compute the time‑averaged update vector, then measure its cosine similarity with the *first principal component* (PC‑1) of the covariance of those averaged updates across nodes.  
   - A similarity close to **+1** means the node’s update is aligned with the dominant low‑rank direction (the “active” sub‑population).  
   - A similarity close to **‑1** means it is anti‑aligned.  
   - Values near **0** indicate orthogonal or random directions.

2. **Frustration‑like L2 penalty** – In the synthetic generator the penalty simply divides every raw update by `(1 + penalty)`. This is an isotropic shrinkage: it reduces the *magnitude* of every update but leaves its *direction* unchanged.

3. **Chimera‑like coexistence hypothesis** – If, as the penalty grows, the system splits into two groups (one that still follows the dominant direction, one that has become inactive or drifts elsewhere), the histogram of similarities will become **bimodal**.  
   - We model that histogram with a **two‑component 1‑D Gaussian mixture** (GMM).  
   - The *weight* of the component whose mean lies near +1 (or the larger‑mean component) is taken as the **active‑population weight**.  
   - A weight that rises from ~0 (all inactive) → ~0.5 (coexistence) → ~1 (all active) as penalty is varied would be the signature of a chimera transition.

### What the current code actually does

```python
raw = np.random.multivariate_normal(np.zeros(dim), C, size=n_nodes)
upd = raw / (1.0 + penalty)          # <-- isotropic scaling
```

* Because the scaling factor is the same for **every** component of every vector, the *direction* of each update is invariant to `penalty`.  
* Consequently:
  - The covariance of the averaged updates, `cov = np.cov(mean_updates, rowvar=False)`, is simply scaled by `1/(1+penalty)^2`.  
  - Scaling a covariance matrix does **not** change its eigenvectors; only the eigenvalues are shrunk.  
  - The PC‑1 (`eigvecs[:, -1]`) is therefore **identical** for all penalty values.  
  - The cosine similarity formula  

    ```python
    sims = (mean_updates @ pc1) / (norms * pc_norm + 1e-12)
    ```

    cancels the common factor `(1+penalty)` in numerator and denominator, leaving `sims` **exactly unchanged** by the penalty.

* In short, with this synthetic generator the alignment histogram **does not depend on penalty at all**. The GMM will return roughly the same two‑component fit (weights ≈ 0.5 / 0.5, means near the true cosine‑similarity distribution) for every penalty value.

### How to obtain a genuine chimera‑like signal

To make the penalty *affect alignment* you need to introduce **anisotropic** or **heterogeneous** effects. A few straightforward modifications:

| Idea | Implementation sketch | What it changes |
|------|-----------------------|-----------------|
| **Penalty‑dependent low‑rank coupling** | `C(p) = Q @ Q.T + p * U @ U.T + 0.1*I` where `U` is another random low‑rank matrix | The covariance’s eigenvectors now rotate with `p`, so PC‑1 shifts and nodes can end up on different sides of the new direction. |
| **Node‑specific penalty** | `penalty_vec = np.random.rand(n_nodes) * p_max` <br> `upd = raw / (1.0 + penalty_vec[:,None])` | Some nodes are shrunk more than others, creating a spread in alignment magnitudes that can produce two clusters. |
| **Stochastic gating (drop‑out‑like)** | With probability `g(p)` set an update to zero; otherwise keep it raw | Creates a literal “inactive” sub‑population (zero updates → similarity undefined → we can treat them as a separate Gaussian centred near 0). |
| **Non‑linear shrinkage** | `upd = raw * np.exp(-penalty * np.linalg.norm(raw,axis=1,keepdims=True)/scale)` | Larger‑norm updates are penalized more, leading to a norm‑dependent direction bias. |

After any of these changes you would re‑run the experiment and look for:

1. **A non‑monotonic trend** in the active‑population weight vs. penalty (low weight at p=0, rise to a peak at intermediate p, decline at high p).  
2. **Clear bimodality** in the similarity histogram at the penalty where the weight is around 0.5 (you can verify with Hartigan’s dip test or by comparing BIC of 1‑ vs. 2‑component GMMs).  
3. **Stability** of the two means: one should stay near +1 (or the dominant direction), the other should drift toward 0 or ‑1, reflecting the “active” vs. “inactive” groups.
