```python
import numpy as np

def layernorm(x, eps=1e-5):
    """LayerNorm applied to the last axis (here the feature dimension)."""
    mean = np.mean(x, axis=-1, keepdims=True)
    var  = np.var (x, axis=-1, keepdims=True)
    return (x - mean) / np.sqrt(var + eps)

# -------------------------------------------------
np.random.seed(42)
dim   = 128
n_samples = 10_000

# Random activations *before* LayerNorm (arbitrary scale)
activations = np.random.randn(n_samples, dim) * 5.0   # shape (n_samples, dim)

# ---- Δa before LayerNorm ------------------------------------------------
# By definition there is *no* change before we apply LN,
# so the “pre‑LN delta” is identically zero.
delta_before = np.zeros_like(activations)   # all zeros
var_before   = np.var(delta_before)         # → 0.0

# ---- Δa after LayerNorm -------------------------------------------------
a_post = layernorm(activations, eps=1e-5)   # shape (n_samples, dim)
delta_after = a_post - activations          # the actual change caused by LN
var_after   = np.var(delta_after)           # scalar variance over all entries

print(f"Var(Δa before LN) = {var_before:.6f}")
print(f"Var(Δa after  LN) = {var_after:.6f}")
```

### Why the numbers look the way they do

| Quantity | Meaning | Expected value (approx.) |
|----------|---------|--------------------------|
| `var_before` | Variance of the *change* before we do anything. Since we defined `delta_before = 0`, the variance is exactly **zero**. | 0 |
| `var_after`  | Variance of the *actual* change introduced by LayerNorm: `Δa = LN(x) – x`. | > 0 (typically ≈ Var(x) + 1 − 2·Cov(LN(x),x)) |

- **LayerNorm does not “shrink” the fluctuation of the activation‑change vector**; instead it *creates* a change whose magnitude is comparable to the original activation scale.  
- Each sample after LN has **zero mean and unit variance** (by construction). The original activations have variance ≈ 5² = 25. Subtracting a zero‑mean, unit‑variance signal from a zero‑mean, variance‑25 signal yields a resultant variance roughly on the order of 25 + 1 (the covariance term reduces it a bit, but not enough to make it smaller than the original variance).  
- Consequently, `Var(Δa after LN)` ends up **larger than zero** and, in practice, **larger than the variance of the original activations** if you look at the per‑dimension variance; the scalar variance over all entries is dominated by the large variance of the pre‑LN activations.

### Interpretation for the “measurement‑induced back‑action” idea

If LayerNorm acted like a quantum‑measurement back‑action that *damps* fluctuations, we would expect the variance of the change vector **after** the operation to be **smaller** than the variance **before** (which is zero in this toy set‑up). The empirical result shows the opposite: the operation introduces a non‑zero change whose variance is substantial. Therefore, in this simple random‑activation test, LayerNorm behaves more like a **re‑scaling and recentering** step that *adds* a deterministic transformation rather than a stochastic measurement that suppresses noise.

If you wanted to test a genuine noise‑reduction effect, you would need to compare the variance of the activations **before** and **after** LN (not the variance of the change). In that case you would see:

```python
var_activations_before = np.var(activations)   # ≈ 25
var_activations_after  = np.var(a_post)       # ≈ 1 (by design)
```

which confirms that LN reduces the *activation* variance (to ~1) while preserving the *relative* structure across features.
