## 1.  What the detection test does

```python
def detect_comb(signal, f0, fs, threshold_factor=3.0):
    """
    Return True if the power at each harmonic (integer multiples of f0)
    exceeds `threshold_factor` times an estimate of the noise floor.

    Parameters
    ----------
    signal : 1‑D np.ndarray
        Real‑valued time‑domain samples.
    f0 : float
        Fundamental spacing of the comb (Hz).
    fs : float
        Sampling rate (Hz).  Must satisfy f0 < fs/2.
    threshold_factor : float, optional
        How many times the noise floor a peak must be to be counted.
        Default = 3 (≈ 5 σ for a Gaussian noise floor).

    Returns
    -------
    bool
        True  → comb detected
        False → not detected
    """
    N = len(signal)
    fft = np.fft.rfft(signal)
    freqs = np.fft.rfftfreq(N, d=1/fs)
    power = np.abs(fft)**2

    # ---- noise floor -------------------------------------------------
    # For each harmonic we mask out a small window (±5 % of the harmonic)
    # and take the median of the remaining power bins.
    noise_bins = []
    for h in np.arange(f0, fs/2, f0):
        # indices of bins that belong to the harmonic ±5 %
        bw = 0.05 * h
        lo = h - bw
        hi = h + bw
        mask = (freqs >= lo) & (freqs <= hi)
        noise_bins.append(power[~mask])   # everything *outside* the window
    noise_floor = np.median(np.concatenate(noise_bins)) if noise_bins else np.median(power)

    # ---- peak test ---------------------------------------------------
    for h in np.arange(f0, fs/2, f0):
        idx = np.argmin(np.abs(freqs - h))
        if power[idx] <= threshold_factor * noise_floor:
            return False          # at least one harmonic too weak
    return True                   # all harmonics passed the test
```

*The function is deliberately simple:* it does **not** try to fit a model, it just checks that every expected harmonic sticks out above a noise floor estimated from the surrounding spectrum.  This matches the “FFT‑based peak test” you described.

## 2.  Signal model

A comb with `K` visible harmonics (up to the Nyquist frequency) is built as

```
x[n] = A * Σ_{k=1}^{K} cos(2π * k * f0 * n / fs)  +  w[n]
```

where `w[n] ~ N(0, σ²)` is white Gaussian noise.  
The **amplitude** `A` controls the SNR:

```
SNR (linear) = (A²/2) / σ²          # power of one sinusoid vs. noise power
```

In the script we parameterise the SNR in dB and convert it to `A` for a chosen `σ = 1` (unit‑variance noise).  Changing `σ` merely rescales `A`; the scaling laws we derive are independent of the absolute noise level.

## 3.  Searching for the minimal `N`

For each combination of `(f0, SNR_dB)` we:

1. Loop over candidate sample sizes `N` (powers of two are convenient for FFT).
2. For each `N` run `n_trials` independent Monte‑Carlo realizations of the noisy comb.
3. Compute the empirical detection probability `p_det = (#detected) / n_trials`.
4. The first `N` with `p_det ≥ 0.9` is recorded as `N_req`.

Because the detection test is monotonic in `N` (more samples → finer frequency resolution → higher peak SNR), a simple linear scan works fine.  If you want to be faster you could replace it with a binary search, but the overhead is negligible for the modest parameter ranges shown below.
