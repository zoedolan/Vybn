We will then test whether higher subspace‑fit predicts lower restitution error across a range of adapter ranks, indicating that the update lives in a low‑dimensional, reversible manifold.

```python
import torch
import torch.nn as nn
from torch.utils.data import DataLoader, TensorDataset

# --- dummy data -------------------------------------------------
# binary classification on random vectors
torch.manual_seed(0)
X = torch.randn(200, 10)
y = (X.sum(dim=1) > 0).long()
train_loader = DataLoader(TensorDataset(X, y), batch_size=32, shuffle=True)

# --- base model -------------------------------------------------
class BaseModel(nn.Module):
    def __init__(self, dim_in=10, dim_out=2):
        super().__init__()
        self.linear = nn.Linear(dim_in, dim_out)
    def forward(self, x):
        return self.linear(x)

base = BaseModel()
base.eval()  # we will not update its weights directly

# --- LoRA adapter ------------------------------------------------
class LoRA(nn.Module):
    def __init__(self, in_features, out_features, rank=4):
        super().__init__()
        self.rank = rank
        self.A = nn.Parameter(torch.randn(out_features, rank) * 0.01)
        self.B = nn.Parameter(torch.randn(rank, in_features) * 0.01)
        self.scaling = 1.0 / rank
    def forward(self, x):
        # x: (batch, in_features)
        return (x @ self.B.t() @ self.A.t()) * self.scaling

def adapt_model(base_model, lora):
    """Returns a function that applies base + LoRA."""
    def fwd(x):
        return base_model(x) + lora(x)
    return fwd

# --- training loop ------------------------------------------------
rank = 4
lora = LoRA(in_features=10, out_features=2, rank=rank)
optimizer = torch.optim.Adam(lora.parameters(), lr=1e-2)
criterion = nn.CrossEntropyLoss()

for epoch in range(20):
    for xb, yb in train_loader:
        logits = adapt_model(base, lora)(xb)
        loss = criterion(logits, yb)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

# --- compute Δ (LoRA weight update) -------------------------------
# Δ = lora.A @ lora.B  (shape out_features x in_features)
with torch.no_grad():
    delta = lora.A @ lora.B  # (2, 10)

# --- subspace‑fit (fraction of energy in top‑r singular values) --
U, S, Vt = torch.linalg.svd(delta, full_matrices=False)
total_energy = (S ** 2).sum()
# we already asked for top‑r where r = rank of LoRA
r = rank
subspace_fit = (S[:r] ** 2).sum() / total_energy

# --- behavioral restitution ---------------------------------------
with torch.no_grad():
    base_out = base(X)                     # (N, 2)
    lora_out = lora(X)                     # (N, 2)
    neg_lora_out = -lora_out               # applying -Δ
    # output after applying -Δ to base model
    out_after_neg = base_out + neg_lora_out
    # mean absolute difference between base_out and out_after_neg
    restitution = torch.abs(base_out - out_after_neg).mean().item()

print(f"Subspace‑fit (top {r} SVs): {subspace_fit:.3f}")
print(f"Behavioral restitution: {restitution:.5f}")
```

### What we expect to see

1. **Subspace‑fit**  
   - By construction, `Δ = A @ B` has rank ≤ `r`.  
   - In exact arithmetic (ignoring the tiny random initialization noise) the SVD will yield **exactly** `r` non‑zero singular values (or fewer if `A` or `B` happen to be rank‑deficient).  
   - Consequently `(S[:r]**2).sum() / (S**2).sum()` will be **≈ 1.0** (often 0.999 + depending on floating‑point round‑off).  
   - If we deliberately increase the LoRA rank beyond the intrinsic rank needed to solve the task, the extra dimensions will capture only noise, and the subspace‑fit will drop slightly below 1.

2. **Behavioral restitution**  
   - Restitution = mean |`lora_out`| measures how large the LoRA correction is on average.  
   - Because the base model is random, the LoRA must learn a mapping that converts the random features into the correct class logits. After training, the LoRA output will be of a magnitude comparable to the logits needed to push the predictions across the decision boundary.  
   - If we increase the LoRA rank, the model can represent the same function with **smaller individual entries** in `A` and `B` (the work can be spread over more dimensions). This typically reduces the average absolute LoRA output, thus **lowering restitution**.  
   - Conversely, a very low rank (e.g., `r=1`) forces the LoRA to pack the entire correction into a single direction, often requiring larger entries in `A` and `B`, which raises the average magnitude of `lora_out` and therefore restitution.

3. **Predicted relationship**  
   - Across a sweep of ranks (say `r = 1, 2, 4, 8`), we should observe:  
     - **Subspace‑fit** starts low for `r=1` (if the true solution needs more than one direction) and climbs toward 1 as `r` reaches the intrinsic dimensionality of the solution.  
     - **Restitution** is highest at the lowest rank (large correction needed in a tight subspace) and decreases as rank grows, plateauing once the rank exceeds what’s needed.  
   - Hence, **higher subspace‑fit (i.e., the update living in a lower‑dimensional subspace) should correlate with lower restitution error**, exactly as the probe hypothesizes.
