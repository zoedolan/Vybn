import numpy as np

# ---------- Synthetic linear bandit ----------
def make_bandit(dim=4, n_arms=5, n_samples=2000, seed=0):
    rng = np.random.default_rng(seed)
    theta_star = rng.normal(size=dim)          # true reward vector
    contexts = rng.normal(size=(n_samples, dim))
    # optimal arm under linear reward: argmax_a <context, theta_a>
    # For simplicity, each arm has its own theta_a = theta_star + noise
    thetas = theta_star + rng.normal(scale=0.1, size=(n_arms, dim))
    rewards = np.einsum('sd,ad->sa', contexts, thetas)   # shape (n_samples, n_arms)
    optimal_arms = np.argmax(rewards, axis=1)
    return torch.FloatTensor(contexts), torch.LongTensor(optimal_arms), thetas

## 2. Why the current architecture is a potential bottleneck  

1. **Seq_len = 1 → attention does nothing**  
   With a single token, the attention weights are always `[1]`. The layer therefore collapses to two linear transformations (`W_v` followed by the readout weight matrix). The overall function is `logits = x·M` for some fixed matrix `M∈ℝ^{d×n_arms}`.  
   *Consequences*:  
   - The model can only represent **linear scoring functions** (no interaction between dimensions beyond a fixed linear map).  
   - Any policy that requires a **context‑dependent non‑linear term** (e.g., the √‑term in LinUCB) cannot be expressed exactly.

2. **Exploration bonus in LinUCB is non‑linear**  
   The term `α·√(xᵀA⁻¹x)` is a scaled Euclidean norm of the context under the metric `A⁻¹`. This is a **convex, homogeneous‑of‑degree‑1** function, but it is not linear unless `A⁻¹` is proportional to the identity and we drop the square‑root (i.e., `α=0`).  
   - If you set `α=0` (pure greedy LinUCB), the target policy reduces to `argmax_a xᵀθ̂_a`, which *is* linear and therefore recoverable by a linear model.  
   - For any `α>0`, the optimal LinUCB arm can differ from the greedy arm, and a linear scorer cannot reproduce those differences for all contexts.

3. **Fisher weighting does not add expressivity**  
   The loss merely re‑weights the contribution of each example according to an estimate of the Fisher information. It can affect *which* linear solution the optimizer finds (e.g., biasing toward low‑variance directions), but it cannot enable the model to represent functions outside the linear span of its inputs.
