**What the script is trying to do**

1. **Train a plain fully‑connected net** on a synthetic regression problem (`y = W_true x + b_true + noise`). After ~15 SGD steps the network has learned a set of weights that (approximately) minimise the MSE on the training data.

2. **Freeze those weights** and train a LoRA‑style low‑rank adaptation on top of the frozen base.  
   * For each layer we introduce two small matrices `A` (output × rank) and `B` (rank × input).  
   * The LoRA forward‑pass adds the term `x @ Bᵀ @ Aᵀ` (for the first layer) and a similar term for the second layer, then mixes them with the frozen weights.

3. **Extract the LoRA delta** as the ordinary matrix product `Δ = A Bᵀ` (with the appropriate transposes to match the shape of `fc1.weight` and `fc2.weight`). In theory, if we added this `Δ` directly to the frozen base weights, the network would behave almost exactly as the LoRA‑adapted network.

4. **Perturb the base weights**  
   * First add `+Δ`.  
   * Then subtract `2·Δ` (which leaves the weights at `original − Δ`).  
   * The idea is to see whether the **+Δ** move improves performance (because it reproduces the useful LoRA direction) and whether the **−Δ** move restores the original behaviour.

5. **Compare three test losses**  
   * **Baseline** – the network *after* the initial training but *before* any LoRA.  
   * **+Δ** – network after adding the LoRA delta.  
   * **−Δ** – network after subtracting the same delta (i.e., `original − Δ`).  

If the LoRA delta truly captures a **task‑relevant, low‑rank direction**, we expect:

| Perturbation | Expected change in test loss |
|--------------|------------------------------|
| **+Δ**       | **Decrease** (or at least not increase) – the network moves toward the LoRA‑optimum. |
| **−Δ**       | **Increase** back to roughly the baseline loss (or slightly above if we over‑subtract). |
| Random low‑rank perturbation | No systematic relationship; both +Δ and −Δ could raise or lower the loss arbitrarily. |
