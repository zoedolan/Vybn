## Result
**Analytical response**

The experiment can be understood in terms of how the parameters θ of the network move during the two post‑B‑training phases.

1. **State after pretraining on Task A**  
   After 5 epochs on A we have a parameter vector θ₀ that (approximately) minimizes the empirical risk  
   \[
   \mathcal{L}_A(θ)=\frac{1}{|D_A|}\sum_{(x,y)\in D_A}\ell(f_θ(x),y)
   \]
   where f_θ is the MLP. The representation h_θ(x)=g_θ(x) (output of the penultimate layer) is thus arranged so that a linear probe w can separate the two classes of A with high accuracy.

2. **Training on Task B (common to both conditions)**  
   The next 5 epochs perform stochastic gradient descent on \(\mathcal{L}_B\):
   \[
   θ_{k+1}=θ_k-η∇\mathcal{L}_B(θ_k)+\xi_k,
   \]
   where ξₖ denotes the noise from mini‑batching. After these steps we obtain θ₁.  
   The change in representation for a fixed input x can be linearised (first‑order Taylor):
   \[
   h_{θ₁}(x)≈h_{θ₀}(x)+J_h(θ₀;x)·(θ₁-θ₀),
   \]
   with J_h the Jacobian of the penultimate‑layer features w.r.t. θ. Hence the drift of the feature space is proportional to the parameter update Δθ = θ₁‑θ₀ = ‑η∑∇ℬ.

3. **Condition (i): hiatus**  
   During the hiatus we set the optimizer to “disabled” and only run forward passes, i.e. we keep θ fixed at θ₁. Consequently,
   \[
   θ_{\text{hiatus}}=θ₁,\qquad Δθ_{\text{hiatus}}=0,
   \]
   and the feature map stays exactly h_{θ₁}(·). The linear probe trained on frozen features therefore sees the same representation that existed after the B‑training phase. Any change in probe accuracy relative to the pretrain level can only come from the *implicit* effect of the B‑updates that produced θ₁.

4. **Condition (ii): continuous training on B**  
   Here we continue SGD on \(\mathcal{L}_B\) for another 5 epochs, obtaining
   \[
   θ_{\text{cont}}=θ₁-η∑_{t=1}^{5}\nabla\mathcal{L}_B(θ_t)+\text{noise}.
   \]
   The total parameter shift from the pretrained state is
   \[
   Δθ_{\text{cont}}=(θ₁-θ₀)+(θ_{\text{cont}}-θ₁)= -η\Big(\sum_{k=0}^{4}\nabla\mathcal{L}_B(θ_k)+\sum_{t=1}^{5}\nabla\mathcal{L}_B(θ_t)\Big).
   \]
   The induced feature drift is
   \[
   Δh_{\text{cont}}(x)≈J_h(θ₀;x)·Δθ_{\text{cont}}.
   \]
   Because the linear probe’s decision boundary is wᵀh, the change in probe logits is approximately
   \[
   Δ\logit≈wᵀJ_h(θ₀;x)·Δθ_{\text{cont}}.
   \]
   Unless the gradients of \(\mathcal{L}_B\) happen to lie in the null‑space of wᵀJ_h (i.e. they do not affect the directions that the probe uses), this term will be non‑zero and will typically reduce the alignment between w and the features of A, lowering probe accuracy.

5. **Expected outcome**  
   - **Hiatus:** feature drift after the hiatus is zero; probe accuracy should stay at the level obtained immediately after the B‑training phase (which is usually close to the pretrain accuracy if B‑training did not already destroy A‑information).  
   - **Continuous B training:** additional updates move the representation further away from the A‑optimal subspace, so the linear probe’s accuracy is expected to drop (or at best stay the same if the two tasks happen to be perfectly aligned).  

   Therefore, analytically we predict  
   \[
   \text{Acc}_{A}^{\text{hiatus}} \ge \text{Acc}_{A}^{\text{continuous}}.
   \]

---

**Intuitive response**

Think of the network after it has learned Task A as a set of “detectors” that fire in just the right way to tell the two A‑classes apart. When we start learning Task B we begin to tweak those detectors so they also become useful for B.  

If we then **take a break** (the hiatus), we stop tweaking the detectors altogether. They stay exactly as they were after the B‑learning phase—no further drift, no extra overwriting. The linear probe, which simply reads off those detectors, therefore sees almost the same pattern it saw before the break, so its performance on A stays high.

If we **keep training on B** without a break, we keep turning the knobs on those detectors. Each tweak moves them a little bit toward being better for B, and unless B happens to need exactly the same pattern as A, the tweaks will start to scramble the pattern that was useful for A. The probe, still wired to read the old pattern, now gets a scrambled signal and its accuracy drops.

In everyday language: a hiatus is like letting the network “settle” after it has learned something useful; continuing to train is like keep‑editing the same notes, which eventually makes the original melody harder to recognize. Hence we expect the hiatus condition to preserve the probe’s accuracy on Task A better than the continuous‑training condition.
