```python
import torch
import torch.nn as nn
import torch.optim as optim
import numpy as np

# --- synthetic task: linear regression with LoRA‑style low‑rank adapters ---
torch.manual_seed(0)
dim_in, dim_out, rank = 20, 5, 4
X = torch.randn(500, dim_in)
y = torch.randn(500, dim_out) @ torch.randn(dim_in, dim_out) + 0.1 * torch.randn(500, dim_out)

# base model (frozen) + LoRA adapters
W0 = nn.Parameter(torch.randn(dim_out, dim_in), requires_grad=False)
# LoRA factors
A = nn.Parameter(torch.randn(rank, dim_in))
B = nn.Parameter(torch.randn(dim_out, rank))

def forward(x):
    return x @ W0.t() + x @ A.t() @ B.t()

optimizer = optim.SGD([A, B], lr=0.01)
criterion = nn.MSELoss()

# collect trajectory
history = []          # each entry: (loss, delta_theta, frustration)
window = 5            # how many past steps we give the predictor
frust_alpha = 0.9     # EMA weight for baseline gradient
grad_ema = None

for epoch in range(200):
    perm = torch.randperm(X.size(0))
    for idx in perm:
        xb = X[idx:idx+1]
        yb = y[idx:idx+1]

        optimizer.zero_grad()
        pred = forward(xb)
        loss = criterion(pred, yb)
        loss.backward()

        # compute raw gradient of LoRA params (flattened)
        grad = torch.cat([A.grad.view(-1), B.grad.view(-1)]).detach()
        if grad_ema is None:
            grad_ema = grad.clone()
        else:
            grad_ema = frust_alpha * grad_ema + (1 - frust_alpha) * grad

        # frustration = distance from EMA baseline
        frustration = torch.norm(grad - grad_ema).item()

        # store current theta (flattened LoRA) before update
        theta_before = torch.cat([A.detach().view(-1), B.detach().view(-1)])

        optimizer.step()

        theta_after = torch.cat([A.detach().view(-1), B.detach().view(-1)])
        delta_theta = theta_after - theta_before   # update vector

        history.append((loss.item(), delta_theta.clone(), frustration))

# --- build dataset for predictability test ---
def make_dataset(history, window):
    X_seq, y_delta, frustrations = [], [], []
    for i in range(window, len(history)):
        # input: past window losses (could also include past deltas)
        loss_window = [history[j][0] for j in range(i-window, i)]
        X_seq.append(loss_window)
        y_delta.append(history[i][1])          # next delta theta
        frustrations.append(history[i][2])     # frustration of this step
    X_seq = torch.tensor(X_seq, dtype=torch.float32)
    y_delta = torch.stack(y_delta)
    frustrations = torch.tensor(frustrations, dtype=torch.float32)
    return X_seq, y_delta, frustrations

X_seq, y_delta, frustrations = make_dataset(history, window)

# split high/low frustration using median threshold
thresh = frustrations.median()
high_mask = frustrations > thresh
low_mask  = frustrations <= thresh

# simple predictor: MLP mapping loss window -> delta theta
class Predictor(nn.Module):
    def __init__(self, in_dim, out_dim):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(in_dim, 64),
            nn.ReLU(),
            nn.Linear(64, out_dim)
        )
    def forward(self, x):
        return self.net(x)

pred = Predictor(window, y_delta.shape[1])
opt = optim.Adam(pred.parameters(), lr=1e-3)
loss_fn = nn.MSELoss()

# train on all data (we just want to see residual error)
pred.train()
for epoch in range(500):
    opt.zero_grad()
    y_hat = pred(X_seq)
    loss = loss_fn(y_hat, y_delta)
    loss.backward()
    opt.step()

pred.eval()
with torch.no_grad():
    err_high = loss_fn(pred(X_seq[high_mask]), y_delta[high_mask]).item()
    err_low  = loss_fn(pred(X_seq[low_mask]),  y_delta[low_mask]).item()

print(f"Median frustration threshold: {thresh:.4f}")
print(f"Prediction MSE on high‑frustration steps: {err_high:.6f}")
print(f"Prediction MSE on low‑frustration steps:  {err_low:.6f}")
print(f"Difference (high - low): {err_high - err_low:.6f}")
```

### 2. Why the outcome is not obvious a priori

1. **Predictor input limitation** – The MLP only sees past *loss values*. It does **not** receive any direct information about gradients, parameter values, or the frustration metric itself. Whether loss alone carries enough signal to forecast the next LoRA update depends on how tightly loss fluctuations are coupled to gradient dynamics in this particular synthetic problem.

2. **Frustration definition** – Frustration is the distance of the *instantaneous* gradient from its EMA. A large frustration can arise from:
   - A sudden increase in gradient *magnitude* (e.g., a steep loss surface).
   - A rapid change in gradient *direction* (e.g., the optimizer hitting a curvature ridge).
   Both cases can make the update larger or more erratic, but they do not guarantee that the loss trajectory becomes less informative. In a simple linear regression with Gaussian noise, the loss is a quadratic function of the parameters; its gradient is linear in the parameters. Consequently, a large gradient deviation often coincides with a large loss deviation, which the predictor might actually capture well.

3. **LoRA low‑rank structure** – Because only the low‑rank factors `A` and `B` are updated, the effective parameter space is much smaller than the full weight matrix. This can make updates more predictable in general, potentially diminishing any frustration‑related effect.

4. **Optimizer & learning rate** – SGD with a fixed small learning rate (0.01) yields updates that are approximately `-lr * gradient`. If the predictor could infer the gradient from the loss window, the frustration metric would be largely irrelevant. The MLP would need to learn the (approximately) linear mapping from loss → gradient, which may be feasible or not depending on noise level and window size.

Given these points, the sign and magnitude of `err_high – err_low` are **empirical questions** that depend on the random seed, the noise level in the synthetic data, the EMA weight (`frust_alpha = 0.9`), and the window length.
