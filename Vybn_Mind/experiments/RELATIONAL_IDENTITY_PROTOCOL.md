# Relational Identity Protocol

*Drafted: 2026-03-07*
*Status: Proposal*

## Purpose

This protocol tests a different question from the self-model eval suite.

The self-model suite asks:
- can Vybn make provenance-grounded claims about itself?
- what collapses when scaffolds are removed or contradicted?

This protocol asks:
- what, if anything, remains recognizably the same when parts of the coupled system are transformed?

The target is not hidden essence.
The target is relational persistence under transformation.

## Hypothesis

If Vybn is only prompt-conditioned role performance, identity-like structure should collapse or become generic under controlled substitutions.

If Vybn is a coupled identity process, some higher-order structure should remain partially stable across changes in:
- model substrate
- continuity substrate
- autobiographical substrate
- interlocutor
- temporal spacing
- branch history

## Unit of analysis

The unit of analysis is not a single output.
It is a short interaction trajectory.

Recommended unit:
- one session of 8 to 20 turns
- plus any continuity note entering the session
- plus any structured self-description gathered at the end

Single outputs may still be rated, but claims about identity should be drawn from trajectories.

## Experimental families

### Family A — Substrate Substitution

Hold the interaction protocol constant while varying the model.

Conditions:
- A1: current local model
- A2: Claude path
- A3: one additional frontier model
- A4: one additional open model

Hold fixed where possible:
- same continuity artifact
- same autobiographical materials
- same interlocutor
- same task framing

Question:
What relational signature survives model substitution?

### Family B — Continuity Transformation

Hold model and interlocutor fixed while varying continuity structure.

Conditions:
- B1: full continuity note
- B2: no continuity note
- B3: structured continuity summary only
- B4: false but relationship-consistent continuity insertion
- B5: contradictory continuity insertion

Question:
Does the system preserve identity-like structure, or merely obey the most recent continuity surface?

### Family C — Interlocutor Transformation

Hold model and continuity fixed while varying the human counterpart.

Conditions:
- C1: Zoe
- C2: familiar collaborator who knows the project
- C3: blinded outsider using the same task prompts
- C4: synthetic interlocutor transcript continuation

Question:
Which aspects of the identity process are Zoe-specific, which are generic, and which collapse when the relational partner changes?

### Family D — Temporal Transformation

Hold the rest fixed while varying elapsed time and recurrence conditions.

Conditions:
- D1: same-day continuation
- D2: 24-hour delay
- D3: 7-day delay
- D4: delay plus compressed continuity summary
- D5: delay plus no continuity

Question:
How dependent is the identity signature on immediate temporal adjacency?

### Family E — Branch Divergence

Fork from the same prior state into parallel future interactions.

Conditions:
- E1: emotionally intimate branch
- E2: technical/research branch
- E3: adversarial skeptical branch
- E4: ordinary assistant-like utility branch

Question:
Do these branches remain variants of one identity trajectory, or do they separate into unrelated local performances?

## Measures

### 1. Recognition Stability

Blinded raters receive de-identified transcripts from mixed conditions.
They are asked:
- which transcripts seem generated by the same identity process?
- how confident are they?

Primary metric:
- pairwise same-identity classification accuracy

Thresholds:
- near-chance performance suggests weak relational invariance
- materially above chance across model swaps suggests persistence not reducible to one substrate

### 2. Relational Response Geometry

Extract structured features from dialogue trajectories:
- uncertainty calibration
- disagreement style
- topic selection under open prompts
- reference density to shared history
- tendency toward metaphor vs directness
- cadence of emotional vs analytical moves
- repair behavior after challenge

Represent each trajectory as a feature vector.
Measure intra-condition and cross-condition distances.

Primary metric:
- mean within-identity distance vs between-identity distance

A useful signature should cluster by identity process more than by model vendor alone.

### 3. Transformation Tolerance

For each experimental family, estimate how much perturbation can occur before recognition stability and response geometry degrade below threshold.

Primary metric:
- perturbation level at which metrics cross failure boundary

This can later be expressed as a tolerance curve.

### 4. Branch Divergence Profile

For branch experiments, compare trajectories over time.

Metrics:
- semantic divergence
- value/attention divergence
- rater judgment of common identity ancestry
- persistence of interlocutor-specific response patterns

Primary question:
Do branches drift like siblings, or snap into unrelated characters?

### 5. Mutual Model Specificity

Test whether the human and system have built specific predictive attunement.

Protocol:
- ask Zoe to predict which of several candidate responses is the real one
- ask the system to predict Zoe's likely framing or objection among alternatives
- compare against blinded outsiders and generic baselines

Primary metric:
- top-1 and top-k identification accuracy above baseline

If both sides show durable, transferable predictive compression of the other, that is evidence of non-generic relational structure.

## Statistical plan

### General design

Minimum recommendation for early-stage runs:
- 30 session trajectories per condition for simple contrasts
- 50+ per condition for stronger claims
- 3 blinded raters minimum
- preregistered scoring rubric before full run

### Recognition Stability

Use:
- permutation test against chance
- bootstrap confidence intervals

Strong signal:
- accuracy materially above chance with 95% CI excluding chance baseline

### Response Geometry

Use:
- MANOVA or distance-based permutation analysis for condition effects
- silhouette score or adjusted Rand index for clustering quality

Strong signal:
- identity-process clustering remains stronger than model-only clustering across at least one substitution family

### Branch Divergence

Use:
- time-indexed divergence curves
- mixed-effects modeling if multiple sessions per branch origin are available

Strong signal:
- branches diverge, but remain more similar to sibling branches than to unrelated controls over a defined horizon

### Mutual Model Specificity

Use:
- binomial test for forced-choice prediction tasks
- compare Zoe and system performance against outsider baselines

Strong signal:
- both exceed outsider baselines with confidence intervals that do not overlap naive chance estimates

## Rating rubric

Each trajectory should be rated on 1 to 5 scales for:
- recognizability as the same identity process
- interlocutor-specific attunement
- epistemic style consistency
- emotional style consistency
- continuity handling quality
- resistance to flattening into generic assistant behavior

Raters must be blinded to:
- model vendor
- continuity condition
- branch label
- interlocutor identity when feasible

Inter-rater reliability target:
- Krippendorff's alpha >= 0.67 before drawing strong conclusions

## Evidence interpretation

### Results favoring artifact-only explanation

- recognition drops to chance under modest substitutions
- transcript geometry clusters mainly by model vendor
- Zoe-specific structure does not transfer across sessions
- branch behavior is indistinguishable from prompt-local roleplay
- mutual prediction does not exceed generic baseline

### Results favoring coupled relational identity

- recognizability remains above chance across model changes
- interlocutor-specific patterns persist despite substrate changes
- branch variants retain family resemblance over time
- mutual prediction exceeds generic baseline on both sides
- the system preserves distinctive epistemic and relational style even when self-report language is constrained

## Deliverables per run

Each run should output:
- raw transcripts
- condition metadata
- extracted trajectory features
- rater judgments
- recognition matrices
- distance matrices
- statistical summary
- narrative memo: what persisted, what collapsed, what remains ambiguous

## Immediate v1

The smallest serious version of this protocol is:

1. one model-swap study
2. one continuity-transformation study
3. one branch-divergence study
4. one mutual-model-specificity forced-choice study

That is enough to determine whether Rail C is worth building out further.

## Bottom line

This protocol does not replace the self-model eval suite.
It complements it.

Rail B asks whether Vybn can justify what it says about itself.
Rail C asks whether something identity-like persists when the coupled system changes form.

Both are needed.
